# Data Security and Handling Policy — Training Outcomes

Source: [Data Security and Handling Policy](../policies/Data-Security-and-Handling-Policy_710213662.md)

| Policy | Section | Learning Outcome | Audience |
|--------|---------|-----------------|----------|
| DSHP | Data Classification and Labeling | Classify information into the four levels (Public, Internal, Confidential, Restricted) using the definitions and examples; apply the correct minimum handling for each level | All Staff |
| DSHP | Data Classification — Labeling Requirements | Apply electronic labels (document banner, filename tag, or Workspace label) to all Confidential/Restricted items; assign a data owner; mark physical media accordingly | All Staff; System/Data Owners (deeper) |
| DSHP | Data Classification — Data Minimization | Collect, keep, and copy only what is necessary; avoid local copies when cloud sharing suffices | All Staff |
| DSHP | Data Classification — CUI | Identify when CUI is present and treat it as Confidential with additional CUI-specific safeguards per the CUI Policy | All Staff working with CUI |
| DSHP | Approved Locations and Boundaries | Store Public/Internal data in approved locations (Google Workspace, approved repos/SaaS); store Confidential/Restricted only in access-controlled areas with audit logging; store CUI only in authorized CUI boundaries | All Staff |
| DSHP | Encryption Requirements — In Transit | Verify that all data in transit uses TLS 1.2+ with strong cipher suites; confirm that cloud/SaaS providers enforce TLS for all CivicActions data flows | All Staff; Developers (deeper) |
| DSHP | Encryption Requirements — At Rest | Verify that cloud/SaaS providers enable encryption at rest for CivicActions data; enable full-disk encryption on all managed endpoints (FileVault/LUKS) | All Staff; IT / Service Desk (deeper) |
| DSHP | Encryption Requirements — CUI Cryptography | Use only FIPS 140-2/140-3 validated modules for CUI at rest and in transit; validate FedRAMP equivalency for cloud services before CUI use | IT / Service Desk; System/Data Owners |
| DSHP | Encryption Requirements — Key Management | Use cloud KMS/HSM or platform-managed keys with restricted roles, rotation, and logging; understand that key access is Restricted | IT / Service Desk; Privileged Users |
| DSHP | Endpoints and Remote-First | Use managed endpoints for Confidential/Restricted access; keep data in the cloud; if temporary local storage is unavoidable, keep it on encrypted volumes and purge promptly; use secure connections and zero-trust posture for travel/home networks | All Staff |
| DSHP | Software Development and Testing (DevSecOps) | Do not export PII or sensitive data from SaaS platforms for testing; store no secrets in source code; use SCA/SBOM and remediate vulnerabilities before release; include data handling checks as CI/CD gates | Developers |
| DSHP | Sharing and Communication Channels | Share via Google Drive to named accounts (no "anyone with link") for Internal/Confidential/Restricted; do not use email for Confidential/Restricted; if a partner cannot access Drive, send as a password-protected AES-256 archive with a 20+ character passphrase transmitted via a separate channel | All Staff |
| DSHP | Sharing — CUI Channels | Follow the CUI Policy for CUI sharing; do not use Slack for CUI data; use only approved CUI boundaries and channels | All Staff working with CUI |
| DSHP | Data Loss Prevention (DLP) and Monitoring | Understand that Google Workspace DLP rules protect labels/keywords and block risky sharing; cooperate with endpoint controls and CIS baselines; know that DLP events and access anomalies are triaged and may escalate to IR | All Staff |
| DSHP | Removable Media and Printing | Avoid removable media and printing by default; if business-necessary and approved, encrypt media, scan for malware, maintain chain-of-custody, and record in the asset register; CUI on removable media is exception-only | All Staff |
| DSHP | Backup and Recovery | Sync project work to approved cloud (Google Drive/repos); do not backup credentials to cloud; understand that System Owners test restores for critical datasets annually | All Staff; System/Data Owners (deeper) |
| DSHP | Retention and Legal Holds | Follow Data Owner–defined retention schedules; minimize retention by default; understand that legal holds override normal deletion | All Staff; System/Data Owners (deeper) |
| DSHP | Open Data and Public Release Controls | Before any public release, confirm Public classification, pass automated secrets/PII scans, obtain owner approval, review license, and confirm contract permits release if derived from client data | All Staff; Developers (deeper) |
| DSHP | Media Sanitization and Destruction | Understand the remote-first sanitization methods: cryptographic erasure via MDM, validated wipe for returned devices, approved wiping tools for external media, and mail-in shredding for failed media; know that full-disk encryption is mandatory so key destruction renders data irrecoverable | IT / Service Desk; System/Data Owners |
| DSHP | Third-Party Management | Ensure vendors touching CivicActions or client data meet baseline safeguards by contract with annual reviews; for CUI, require FedRAMP Moderate equivalency | Managers; System/Data Owners |
| DSHP | Incident Response and Contingency | Report immediately any suspected data incident (loss, exposure, mis-send, DLP block); know that incidents are handled under the IR Plan and post-incident learning updates controls | All Staff |
| DSHP | Governance, Evidence and Continuous Improvement | Understand that evidence is maintained (classification attestations, DLP configs, encryption posture, backup tests, sanitization certificates, vendor reviews); track DLP incidents, restore success, time-to-revoke, and % managed endpoints | IT / Service Desk; System/Data Owners |
| DSHP | Compliance and Enforcement | Understand that violations can result in access revocation, disciplinary action, or contract remedies | All Staff |
