# AI Usage Policy — Training Outcomes

Source: [AI Usage Policy](../policies/AI-Usage-Policy_582418435.md)

| Policy | Section | Learning Outcome | Audience |
|--------|---------|-----------------|----------|
| AIP | General Requirements | Use only CivicActions-approved generative AI tools with IT-managed accounts; never input sensitive or confidential information into AI tools; ensure human oversight for all AI-generated content (verify accuracy, label internally, peer-review before external sharing) | All Staff |
| AIP | Usage in Software Development | Use GitHub Copilot exclusively for source code creation; carefully review and test AI-generated code for correctness and security; describe AI usage in peer review requests; use other approved tools for non-code technical queries | Developers |
| AIP | Usage on Client Projects | Comply with client-specific AI policies and guidance; never enter CUI into AI tools; ensure AI restrictions are included in project onboarding and wrapper documents | All Staff |
| AIP | Usage in Sales | Do not use AI tools for pricing or original RFP content generation; AI may assist with analysis, review, and content rewriting | Managers; Sales staff |
| AIP | Approved Generative AI Tools | Identify the approved tool categories and licensing requirements: general purpose (Slack Chatbot, Gemini, ChatGPT for Teams), coding (GitHub Copilot), specialist (Grammarly, Paxton AI), service integrations (Zoom, Mural, Confluence/Jira, Greenhouse, Figma), and programmatic APIs (Vertex AI, Azure OpenAI — approval required); know that ChatGPT's Atlas browser is not approved | All Staff |
| AIP | AI Usage Approval | Understand that new AI tools or significant new applications require CTO approval (and client approval for client projects); know that a risk assessment guided by the NIST AI Risk Management Framework is required | All Staff |
| AIP | Usage in Professional Development | Freely explore AI tools and topics for learning purposes, provided only publicly available data is used | All Staff |
